Card deck API demo & reference

Stateless piles: clients keep draw_pile, current_card, discard_pile, and shuffle_seed (from deck create; null = random each shuffle) and POST the full JSON back on draw/shuffle/mill. Set shuffle_seed only on /api/decks/create* — later operations reuse the value on the pile object. Dual-track routes use six pile arrays plus shuffle_seed on the dual object.

Other endpoints (reference)

Not wired as live buttons in this page — use the route catalog above or your own client.

Method Path Body / notes
POST /api/decks/create-single Alias of /api/decks/create. {"deck_names": string[], "shuffle_seed"?: …}
POST /api/decks/create-dual Same create body; response has exploration + item piles and shuffle_seed.
POST /api/dual/draw/exploration
/api/dual/draw/item		 Full dual JSON + optional draw_count (or count). Drawn cards go to *_current_cards.
POST /api/dual/shuffle/exploration
/api/dual/shuffle/item		 Full dual JSON; shuffles that track’s draw pile only.
POST /api/single/peek {"draw_pile": [...], "peek_count"?: n, "count"?: n} — read-only; top = index 0.
POST /api/dual/peek/exploration
/api/dual/peek/item		 Full dual JSON + peek_count only (no count alias). Returns requested, returned, cards.
POST /api/single/mill Full single pile JSON + optional mill_count. Omit mill_count → entire draw → discard; 0 → no-op.
POST /api/dual/mill/exploration
/api/dual/mill/item		 Full dual JSON + optional mill_count (same semantics as single mill).
